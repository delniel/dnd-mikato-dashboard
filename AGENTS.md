# Урумир — личный DnD MGE dashboard

## Команды

- `npm.cmd run dev` — локальная разработка.
- `npm.cmd run lint`, `npm.cmd run test`, `npm.cmd run build` — обязательные проверки.

## Архитектура

- `src/data.ts` — неизменяемые начальные данные из исходного DOCX: персонаж, заклинания, Skills Plus, инвентарь и формы.
- `src/domain.ts` — типы, Zod-схема импорта и чистая игровая логика; её изменения должны сопровождаться тестом в `src/domain.test.ts`.
- `src/store.ts` — Zustand-состояние только для редактируемых данных.
- `src/db.ts` — Dexie/IndexedDB: снимок персонажа и локально загруженные изображения.
- `src/App.tsx` — переиспользуемые представления листа; не создавайте отдельные компоненты на каждое заклинание/умение/предмет.

## Данные и правила

- Добавляйте заклинание, навык, предмет или форму одной записью в `src/data.ts`; карточки и фильтры должны подхватить её автоматически.
- Не заменяйте значения проверок и спасбросков формулами без явного согласования: это кастомная редакция, а не DnD 5e.
- Изображения принадлежат IndexedDB. Не кладите пользовательские загруженные файлы в репозиторий.
- Сохраняйте неясные исходные механики как текстовые заметки, не изобретайте правила.
